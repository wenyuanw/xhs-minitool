# 小红书小工具发布检查清单

在交付新项目、迁移项目或兼容性修复前使用本清单。自动扫描不能替代运行时验证；必须同时完成静态检查和移动端人工测试。

## 一、需求与架构

| 检查项 | 通过标准 |
|---|---|
| 核心功能 | 断网状态下仍能完成主要任务。 |
| 服务端依赖 | 不依赖 API、登录、鉴权、推送、轮询或实时同步。 |
| 计算方案 | 不依赖 WASM、Worker、SharedArrayBuffer 或云端模型。 |
| 导航行为 | 不依赖外链、新窗口、iframe、表单跳转或跨小工具跳转。 |
| 文件行为 | 不依赖任意文件类型、浏览器下载或文件系统访问。 |
| 降级方案 | 每个被移除的能力都有明确的本地替代方案或产品说明。 |

## 二、项目结构

| 检查项 | 通过标准 |
|---|---|
| HTML 入口 | 最终提交包中只有一个 `.html` 文件。 |
| 文件类型 | 只包含 `.html`、`.css`、`.js`、支持的图片、`.woff` / `.woff2` 和 `.json`。 |
| 构建产物 | 不包含 node_modules、源码映射、构建缓存、服务端文件或开发配置。 |
| 路径 | 所有资源使用相对路径，且引用目标存在并位于最终提交目录内。 |
| 符号链接 | 不包含符号链接；需要的资源复制为提交目录内的真实文件。 |
| 外部资源 | 不存在 `http:`、`https:` 或协议相对 URL。 |
| 字体和图标 | 全部本地打包，不引用在线字体或 CDN 图标。 |

## 三、HTML 与 JavaScript

| 检查项 | 通过标准 |
|---|---|
| 脚本加载 | 仅使用 `<script src="…">` 引用包内 `.js`。 |
| 内联脚本 | 不存在 `<script>…</script>`。 |
| 事件绑定 | 不存在 `onclick`、`onchange` 等行内事件；统一使用 `addEventListener`。 |
| 动态执行 | 不存在 `eval`、`new Function`、`javascript:` URI。 |
| 网络 | 不存在 fetch、XHR、WebSocket、EventSource、WebRTC 或远程资源。 |
| 后台能力 | 不存在 Worker、SharedWorker、ServiceWorker、SharedArrayBuffer。 |
| 禁用 API | 不使用定位、剪贴板、硬件连接、传感器、凭据、屏幕共享、新窗口或 prompt。 |
| 下载与嵌入 | 不存在 iframe、object、embed、`download` 属性或 blob 下载。 |
| 表单 | 不通过 `<form>` 提交进行导航；本地表单事件使用 `preventDefault`。 |

## 四、样式与移动端体验

| 检查项 | 通过标准 |
|---|---|
| 视口 | 设置移动端 viewport；不产生无意的横向滚动。 |
| 安全区域 | 顶部和底部关键操作考虑 `env(safe-area-inset-*)`。 |
| 触摸目标 | 主要按钮易于触摸，间距足够，不依赖 hover。 |
| 键盘 | 输入框聚焦和软键盘弹出后，主要操作仍可见。 |
| 屏幕尺寸 | 至少验证窄屏、小屏和较高屏幕。 |
| 文字与对比度 | 关键信息在明暗环境下清晰可读。 |
| 加载与空状态 | 本地初始化、空数据、无权限和错误状态都有反馈。 |

## 五、媒体与存储

| 检查项 | 通过标准 |
|---|---|
| 权限 | 摄像头和麦克风在授权、拒绝和取消时均有处理。 |
| 文件选择 | 只承诺图片和视频，不把 `accept` 当作突破容器限制的手段。 |
| 预览兼容 | 使用 `data:` / `blob:` 图片时，已决定是否兼容 9.37 之前版本。 |
| 数据隔离 | 不依赖其他小工具或外部页面读取当前工具数据。 |
| 数据持久化 | 不向用户承诺永久保存；必要时提供清空和重置能力。 |
| Cookie | 不用于服务端会话或鉴权；本地数据优先使用 localStorage / IndexedDB。 |

## 六、依赖审查

在引入每个第三方库前，检查其发布包而不仅是文档。确认没有动态代码、WASM、Worker、网络请求、远程字体／图片、服务端模块或按需远程加载。若无法证明兼容，优先使用原生 JavaScript 或复制并审查所需的最小实现。

## 七、验证步骤

沿用主技能定义的 `<skill-dir>` 与 `<project-dir>`；执行前将占位符替换为实际绝对路径。

1. 运行 `python3 "<skill-dir>/scripts/validate_xhs_tool.py" "<project-dir>"`。
2. 修复所有 `ERROR`；逐项判断 `WARNING`，不得无说明地忽略。
3. 在断网条件下打开唯一 HTML 入口，验证核心流程。
4. 测试首次进入、已有数据、空数据、无权限、用户取消和异常输入。
5. 测试至少一种实际移动端视口；涉及媒体时验证授权和拒绝流程。
6. 对生成的最终提交目录再次运行校验，不要只检查源码目录。
7. 在交付说明中列出校验结果、人工测试范围和剩余限制。
